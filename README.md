# Food Delivery Web Application (DNICK)

Ова е веб апликација за онлајн нарачување и достава на храна, развиена со **Django framework**, како проект по предметот *Дизајн на интеракцијата човек-компјутер*. Апликацијата е инспирирана од платформи како Korpa, Wolt и Glovo и е имплементирана со јасна модуларна архитектура, role-based пристап и симулирани реални процеси (нарачка, плаќање, достава).

---

## Архитектура

Системот е поделен во повеќе Django апликации, при што секоја апликација има јасно дефинирана одговорност:

- **core** – централна апликација која ги содржи глобалните поставки, базниот темплејт, статичките ресурси и механизмот за иницијално полнење на базата (seed data)
- **accounts** – автентикација, регистрација и управување со кориснички ролји преку custom User модел
- **restaurants** – управување и прикажување на ресторани, менија, категории и работно време
- **orders** – кошничка, креирање на нарачки, пресметка на вкупна цена и следење на статуси
- **payments** – изолирана логика за плаќање со Stripe (test режим и симулирано плаќање)

Оваа модуларна поделба овозможува чиста архитектура, лесно одржување и можност за понатамошно проширување на системот.

---

## Кориснички ролји

Апликацијата користи **role-based access control (RBAC)** модел со следните кориснички ролји:

- **Customer** – пребарува ресторани, прегледува менија, додава производи во кошничка, креира нарачки и врши плаќање  
- **Owner** – управува со свои ресторани и нивните менија  
- **Courier** – има пристап до нарачки наменети за достава  
- **Admin** – целосна контрола врз системот преку админ панел  

Секој нов корисник автоматски добива ролја **customer**.  
Промена на ролја е дозволена **исклучиво за администратор** и се врши преку Django Admin панелот.

---

## Плаќање

Плаќањето е имплементирано преку **Stripe (test mode)** и е целосно изолирано во `payments` модулот.  
За секоја нарачка се креира платежна евиденција, а доколку Stripe е конфигуриран, се иницира Stripe PaymentIntent и се користи Stripe Elements на frontend.

Доколку Stripe не е достапен, апликацијата овозможува **симулирано успешно или неуспешно плаќање**, со идентичен тек како кај реална трансакција. Овој пристап овозможува целосна демонстрација и тестирање без реални финансиски операции.

## Стартување на апликацијата

Апликацијата се стартува со извршување на миграциите и командата:

```bash
python manage.py runserver
```
За пристап до Django Admin панелот потребно е да се креира superuser со командата:

```bash
python manage.py createsuperuser
```
Во системот веќе постојат следните тест корисници:

Customer – user: customer, pass: customer

Admin – user: admin, pass: admin

Courier – user: courier, pass: courier

Owner – user: owner, pass: owner

Забелешка
Податоците за рестораните и менијата се scrap-ани од платформата Korpa и се внесени во базата преку seed_data.py, со цел да се симулира реална продукциска средина без зависност од надворешни API сервиси.
